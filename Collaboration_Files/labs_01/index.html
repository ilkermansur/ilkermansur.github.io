QoS-LAB-02 : Write a policy on BB router with two way (Header and Payload inspection ) respectivly for matching (set dscp46) telnet traffic and try telnet from Cube-1 to Cube-2 router. Copy running config for each scenario.

Cube-1 Initial Configuration


interface GigabitEthernet1.1002
 encapsulation dot1Q 1002
 ip address 142.102.64.151 255.255.255.0
 shutdown
!
interface GigabitEthernet1.1009
 encapsulation dot1Q 1009
 ip address 9.9.9.2 255.255.255.0
!
ip route 142.102.64.0 255.255.255.0 9.9.9.1

BB Initial Configuration


interface GigabitEthernet1.1009
 encapsulation dot1Q 1009
 ip address 9.9.9.1 255.255.255.0

Cube-2 Initial Configuration

ip route 9.9.9.0 255.255.255.0 142.102.64.1

Solution

Configuration on Backbone Router

ip access-list extended TELNET
 permit tcp any any eq telnet
!
class-map match-all TELNET_SETDSCPEF
 match access-group name TELNET
!
policy-map TELNET_SETDSCPEF
 class TELNET_SETDSCPEF
  set dscp ef
!
interface GigabitEthernet1.1002
 service-policy output TELNET_SETDSCPEF

Verification

BackBone_RT#show monitor capture TELNET buffer brief
BackBone_RT#monitor capture TELNET export tftp://142.102.64.101/Telnet.pcap
